# 老人用药记录：一天多次情况记录升级

本版本把“补充记录 → 情况记录”改为真正的多条记录模式。

## 新逻辑

- 一天可以添加多条情况记录。
- 每条记录包含“记录时间 + 内容”。
- **不会自动保存**；只有点击“保存本条记录”后才会进入本机同步队列并上传云端。
- 保存后输入框自动清空，方便继续添加下一条。
- 当天记录会在输入区下方按时间显示。
- 每条记录可修改或删除。
- 输入了内容但尚未保存时，切换日期或离开页面会先询问是否放弃。
- 旧版单条“当天情况”会自动迁移为当天的一条历史记录，不会丢失。

## 部署步骤

### 1. 更新腾讯云 CloudBase 云函数

将 `cloud-function-index.js` 的全部内容复制到 `medication-sync` 云函数的 `index.js`，然后重新部署。

这一步必须先做，因为新版增加了：

- `upsertNote`：新增或修改一条情况记录
- `deleteNote`：删除一条情况记录
- `days[日期].notes`：当天多条记录数组

原有 `saveNote` 仍被保留，用于兼容旧页面。

### 2. 更新 GitHub Pages

用新版 `index.html` 覆盖仓库根目录原文件并提交。

首次检查可使用带参数的网址绕过缓存，例如：

`https://xjl2026.github.io/med/?release=13`

确认无误后继续使用正式网址即可。

## 数据兼容

无需新建数据库集合，也无需修改环境变量。旧数据中的 `day.note` 会自动转换成 `day.notes` 中的一条记录。
