# 老人用药记录

一个面向住院及居家陪护场景的家庭用药记录网页。家人打开手机网页后，可以快速查看下一次需要服用的药、药盒照片、服用时点和剂量，并记录本次执行情况；用药方案只允许管理员维护。

## 在线地址

- 正式页面：<https://xjl2026.github.io/med/>
- GitHub 仓库：<https://github.com/xjl2026/med>

> 本项目用于家属之间同步和记录信息，不提供诊断，也不能替代医生、护士、药师或药品说明书。医嘱发生变化时，应先按最新医嘱执行，再联系建瓴后台修改网页方案。

## 主要功能

- 顶部快捷显示最近一次待处理用药，并展示对应药盒照片。
- 今日用药按照实际服用逻辑排列，例如：早餐前、早餐随餐、早餐后、午餐随餐、晚餐随餐、睡前。
- 支持记录“已服用”“稍后服用”“本次未服”“医生要求停服”等情况。
- 所有家人均可查看“正在服用”和“已停用”的药品清单。
- 支持药盒照片放大查看，帮助不同陪护人员辨认药品。
- 支持补充记录血压、体温、血氧、当天精神状态、睡眠及查房情况。
- 陪护提醒始终展开，进入页面即可看到。
- 普通家人只能查看和记录，不能增加、修改或停用药品。
- 管理员可以调整方案、生效时间、停用药品、恢复药品、备份和导入数据。
- 支持云端同步、离线待上传、失败重试、版本检查和北京时间校准。

## 使用角色

| 角色 | 可以做什么 | 不可以做什么 |
| --- | --- | --- |
| 家人 / 陪护人员 | 查看今日用药、确认已服用、记录其他情况、查看正在服用和已停用药品、填写补充记录 | 新增药品、修改剂量、调整服用时间、停用或删除药品、恢复备份 |
| 管理员 | 拥有家人全部权限，并可维护药品资料和用药方案 | 管理员仍不应自行作出医疗决定，方案应以最新医嘱为依据 |

## 家人使用流程

1. 打开正式页面并输入家庭访问密码。
2. 优先查看页面顶部的“下一次用药”。
3. 对照药名、药盒照片、剂量和服用时点。
4. 已经服下时，点击“确认已服用”。
5. 未按原计划服用时，点击“其他情况”，选择相应状态并填写原因。
6. 在“用药清单”中查看哪些药正在服用、哪些药已经停用。
7. 如医生调整药品、剂量、次数或服用时点，请联系建瓴后台修改。

## 状态说明

- **已服用**：本次药物已经实际服下。
- **稍后服用**：暂时未服，之后仍需要继续处理；不会计为已完成。
- **本次未服**：确认本次没有服用，也不再按本次计划补服。
- **医生要求停服**：医生临时取消这一次用药，不代表整种药已经长期停用。
- **已停用药品**：整种药的方案已经结束，只在“用药清单”中保留历史信息，不再生成每日任务。

## 用药文案规范

面向家人的页面尽量使用自然描述，不把时间限定得过死：

- 早餐前约 30 分钟
- 早餐随餐
- 早餐后
- 午餐随餐
- 晚餐随餐
- 睡前

系统内部可以保留参考时间用于排序和判断“下一次用药”，但家人页面优先展示上述服用时点。每种药的说明统一为：

1. 药品名称
2. 用途
3. 服用时点
4. 每次剂量
5. 必要提醒
6. 药盒照片或外观特征

## 技术架构

- **前端**：单文件 `index.html`，部署在 GitHub Pages。
- **后端**：腾讯云 CloudBase 云函数 `medication-sync`。
- **数据库**：CloudBase 集合 `medication_data`。
- **登录方式**：家庭访问密码和独立管理员密码。
- **设备识别**：每台设备自动生成匿名设备编号，不要求陪护人员填写姓名或关系。
- **审计信息**：后台可记录匿名设备编号、操作时间和脱敏后的 IP 哈希，不保存原始 IP。


公开仓库中不要提交以下内容：

- 家庭访问密码或管理员密码
- `FAMILY_KEY`
- GitHub Token、腾讯云密钥等凭据
- 包含真实姓名、病历、联系电话的文件
- 数据库导出、备份 JSON 或初始化患者数据

## CloudBase 配置

云函数需要以下环境变量，README 和前端代码中只保存变量名称，不保存真实值：

| 环境变量 | 用途 |
| --- | --- |
| `ACCESS_PASSWORD` | 家人进入网页使用的共同访问密码 |
| `ADMIN_PASSWORD` | 管理员维护用药方案时使用的独立密码 |
| `FAMILY_KEY` | 后台签发和验证会话使用的随机密钥 |
| `ALLOWED_ORIGIN` | 允许访问后台的前端来源，例如 `https://xjl2026.github.io` |

建议数据库集合权限保持为仅后台可读写，前端所有数据操作统一通过云函数完成。

## 部署与更新

### 只修改页面时

1. 用新版 `index.html` 覆盖 GitHub 仓库中的旧文件。
2. 提交到 GitHub Pages 使用的分支。
3. 等待 Pages 发布完成。
4. 首次检查时可在网址后临时添加版本参数，例如 `?release=13`，避免手机缓存旧页面。
5. 确认正常后继续使用不带参数的正式网址。

### 修改后台时

1. 先更新 CloudBase 云函数并确认部署成功。
2. 再更新 GitHub Pages 的 `index.html`。
3. 使用普通家人账号测试查看和记录。
4. 使用管理员密码测试药品维护权限。

不要在后台接口尚未部署时提前发布依赖新接口的前端版本。

## 上线检查清单

- 家庭访问密码可以正常登录。
- 普通家人看不到药品新增、调整、停用或删除按钮。
- 顶部“下一次用药”与下方最近一条用药内容一致，并显示正确照片。
- 今日用药排序符合“餐前 → 随餐 → 餐后”的逻辑。
- “确认已服用”和“其他情况”可以保存并同步到其他设备。
- “稍后服用”仍保留为待处理项目。
- “正在服用”和“已停用”清单对所有家人可见。
- 陪护提醒始终展开。
- 跨过北京时间零点后可以自动切换到新一天。
- 断网记录能够进入待上传状态，恢复网络后可以同步。
- 管理员可以进入和退出管理模式，退出后维护按钮立即隐藏。
- 云端备份和恢复功能可用。

## 数据和隐私

这是一个公开部署的网页项目。即使页面有密码，也不应把完整病历、身份证号、住址、电话号码等高敏感信息写入公开仓库。药品照片应尽量裁掉患者姓名、住院号、二维码和处方标签等个人信息。

更换手机或清理浏览器数据后，匿名设备编号可能发生变化，但云端用药记录不会因此丢失。重要数据应定期使用管理员功能进行云端备份或导出保存。

## 医疗安全说明

- 页面中的剂量、频次、服用时点和停药状态必须来自医生医嘱、护士交代、药师指导或药品说明书。
- 不因网页显示内容自行加药、减药、补服、停药或改变剂量。
- “已服用”必须在确认患者实际服下后再记录。
- 对是否漏服、能否补服、药物相互作用、出血、过敏或其他异常存在疑问时，应立即联系医院医护人员。
- 医嘱变化应使用带生效时间的新方案更新，不直接覆盖历史剂量，以便保留原有记录。
