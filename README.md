# 老人用药记录

面向家庭陪护场景的轻量用药记录网页。家人可以通过手机或电脑查看当天用药安排、辨认药盒、记录每次执行结果，并补充身体指标和护理情况；药品方案仅由管理员维护。

> 本项目只用于家属之间的信息同步与执行记录，不能替代医生、护士、药师或药品说明书。药品、剂量、频次和停药状态如有变化，应以最新医嘱为准。

## 功能概览

### 当天用药

- 顶部快捷显示当天列表中**第一条尚未处理的用药**。
- 快捷卡片与下方用药卡片使用同一套药名、药盒照片、剂量和服用说明。
- 当天用药按实际执行顺序排列，例如：早餐前、早餐随餐、早餐后、午餐随餐、晚餐随餐、睡前。
- 药盒照片可放大查看，方便不同陪护人员辨认。

### 用药记录

每次用药可记录以下状态：

| 状态 | 含义 | 是否继续显示为待处理 |
| --- | --- | --- |
| 已服用 | 已确认患者实际服下本次药物 | 否 |
| 稍后服用 | 当前暂缓，稍后仍需处理 | 是 |
| 本次未服 | 本次没有服用，并且不再按本次计划补服 | 否 |
| 医生要求停服 | 医生临时取消本次用药，不代表整种药长期停用 | 否 |

已保存的记录可以修改或撤销。撤销后，该条会按原有用药顺序重新成为待处理项目。

### 补充记录

- 记录血压、心率、体温、血氧、体重等身体指标。
- 一天可添加多条情况记录，例如精神状态、睡眠、查房反馈或其他异常。
- 情况记录不会自动保存，必须点击“保存本条记录”。
- 当天记录可查看、修改和删除。

### 用药清单与陪护提醒

- 所有家庭成员都能查看“正在服用”和“已停用”的药品。
- 已停用药品保留原方案和历史信息，但不再生成当天用药任务。
- 陪护提醒固定展开，进入页面即可查看。
- 医嘱如有变化，请联系管理员更新后台方案。

### 管理员功能

普通家庭成员只能查看和记录，不能修改药品方案。管理员可执行：

- 新增药品；
- 调整剂量、频次、服用时点和生效时间；
- 停用或恢复药品；
- 查看最近操作；
- 备份、导出和恢复数据。

方案调整应创建带生效时间的新版本，避免直接覆盖历史用药信息。

## 使用方法

1. 打开应用地址并输入家庭访问密码。
2. 优先查看顶部快捷卡片，确认药名、药盒照片、剂量和服用时点。
3. 患者实际服药后，点击“确认已服用”。
4. 遇到暂缓、未服或医生临时取消时，点击“其他情况”选择对应状态。
5. 需要交接的信息可写入“补充记录”。
6. 在“用药清单”中核对哪些药仍在服用、哪些药已经停用。

## 页面规则

### 顶部快捷卡片

顶部卡片不单独计算另一套“下一次用药”。它始终复制下方“今日用药”中第一条尚未处理的记录。

例如：第 1 条和第 3 条已记录、第 2 条未记录时，顶部必须显示第 2 条，而不是跳到第 4 条。

### 服用时间文案

面向家人的页面优先使用容易执行的自然描述，不把时间限定得过死，例如：

- 早餐前约 30 分钟；
- 早餐随餐；
- 早餐后；
- 午餐随餐；
- 晚餐随餐；
- 睡前。

系统内部可以保留参考时间用于排序，但具体执行仍以最新医嘱为准。

## 技术架构

| 模块 | 技术 |
| --- | --- |
| 前端 | 单文件 HTML、CSS、原生 JavaScript |
| 静态托管 | GitHub Pages |
| 后端接口 | 腾讯云 CloudBase 云函数 |
| 数据存储 | CloudBase 文档型数据库 |
| 身份验证 | 家庭访问密码、独立管理员密码 |
| 设备识别 | 浏览器本地生成的匿名设备编号 |

前端通过云函数读写数据，不直接访问数据库。数据库集合建议设置为仅后台可读写。

## 数据同步

- 操作先写入本地状态和待同步队列，再上传云端。
- 网络恢复后会继续尝试同步。
- 页面重新回到前台时会主动拉取最新数据。
- 顶部“已同步”表示当前数据已经成功上传云端。
- 匿名设备编号用于区分操作来源，不要求陪护人员填写姓名或关系。
- 清除浏览器数据、使用无痕模式或更换浏览器后，可能生成新的设备编号。


CloudBase 云函数代码和环境变量应在腾讯云后台管理。若将云函数源码纳入版本控制，建议单独建立目录，并确保不提交任何密码或密钥。

## 部署

### 前端

1. 将最新版 `index.html` 放在仓库根目录。
2. 在 GitHub 仓库设置中启用 GitHub Pages。
3. 选择实际发布分支和根目录。
4. 提交后等待 Pages 完成发布。
5. 首次验证可在地址后临时添加查询参数绕过缓存，例如：

```text
https://xjl2026.github.io/med/?v=发布版本号
```

正式分享时仍使用不带参数的固定地址。

### CloudBase 后端

云函数名称：`medication-sync`

需要配置的环境变量：

| 变量名 | 作用 |
| --- | --- |
| `ACCESS_PASSWORD` | 家庭成员进入应用使用的访问密码 |
| `ADMIN_PASSWORD` | 管理员进入维护模式使用的独立密码 |
| `FAMILY_KEY` | 后台会话签发与验证使用的随机密钥 |
| `ALLOWED_ORIGIN` | 允许调用接口的前端来源 |

环境变量的真实值不得写入 `index.html`、`README.md` 或公开提交记录。

## 更新顺序

- 只修改界面或文案：仅更新 `index.html`。
- 修改接口、数据结构或同步逻辑：先部署 CloudBase 云函数，再更新 `index.html`。
- 发布后应分别测试普通家庭成员权限和管理员权限。

## 隐私与安全

这是公开托管的前端项目。即使页面设置了密码，也不要在仓库中提交：

- 患者姓名、身份证号、住院号、家庭住址或联系电话；
- 完整病历、检查报告或数据库备份；
- 家庭访问密码、管理员密码、`FAMILY_KEY`；
- GitHub Token、腾讯云密钥或其他凭据；
- 带有患者标签、处方二维码等敏感信息的原始照片。

药盒照片应尽量裁掉姓名、住院号、二维码和处方标签。

## 医疗安全声明

- 页面中的药品、剂量、次数、服用时点和停药状态必须来自最新医嘱或专业医护人员指导。
- 不根据网页自行加药、减药、停药、补服或改变剂量。
- 只有确认患者实际服下药物后，才记录为“已服用”。
- 对漏服、补服、相互作用、出血、过敏或其他异常存在疑问时，应立即联系医护人员。
- 网页内容与最新医嘱不一致时，以最新医嘱为准，并尽快由管理员更新方案。

## 维护状态

本项目为家庭内部使用工具，功能会根据实际陪护流程持续调整。提交修改前应优先保证：

1. 用药信息准确；
2. 操作流程简单；
3. 历史记录可追溯；
4. 普通成员无法修改用药方案；
5. 不在公开仓库泄露敏感信息。
